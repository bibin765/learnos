# Learning OS

> **Learning-as-Code.** Curricula are versioned markdown, not bookmarks. Every topic is a folder. Every roadmap, note, and artifact lives in git. Claude helps you practice — it does not read the sources for you.

A personal system for deliberate technical (or any) learning, with a web UI that orchestrates the four-phase weekly loop: **prime → read → explain → defend**. Backed by your local Claude Code session (no API key needed) or the Anthropic API.

## Quick start

```bash
git clone <your-fork-or-origin>.git
cd <repo>/ui
npm install
npm run dev
```

Open <http://localhost:4321>. Default backend is the **local `claude` CLI** — no API key required if you have Claude Code installed. Flip to **API** in the top bar and paste a key if you'd rather go direct to Anthropic.

In dev, the UI reads content from the parent filesystem, so you can iterate without pushing anywhere. To deploy, point `PUBLIC_GITHUB_OWNER` / `PUBLIC_GITHUB_REPO` at the repo's GitHub path and build (`npm run build`).

## The weekly loop

Each topic in `roadmaps/` has a 4-week roadmap. Each week is a card on the topic page with a big **Start studying** button that opens Study mode, a four-phase loop:

| Phase | What you do | What the app does |
|---|---|---|
| **01 Prime** | Read 3 framing questions before you open the source | Generates questions tailored to the week's concepts |
| **02 Read** | Engage the primary sources; capture notes and confusions | Lists sources with search bridges; autosaves your notes |
| **03 Explain** | Write a 200+ word plain-language explanation | Streams a sentence-by-sentence gap report (✅ ⚠️ ❌ 🕳️) |
| **04 Defend** | Let the Socratic tutor grill you on the cracks | Multi-turn chat that refuses to hand over answers |

Progress dots on each week card show which phases you've completed. Sessions persist across refreshes.

## The three templates

All three route through whichever backend you've picked (CLI or API) and stream output live.

### 🧭 Roadmap wizard — `/templates/roadmap-gen`

Two phases. Phase 1: Claude asks 3–5 topic-specific clarifying questions that discriminate your level and goal (not "what's your level?" — "have you written `async fn` returning `impl Future`?"). Phase 2: generates a dense, primary-sources-only 4-week roadmap, tailored to your answers. One click saves it to `roadmaps/<slug>/` and seeds a progress log.

### 🗨️ Socratic tutor — `/templates/socratic`

Multi-turn chat. You state a concept; the tutor refuses to hand over the answer, asks one discriminating question at a time, steelmans your position before challenging it, and names your moves when you hand-wave. Escape hatches: `direct answer please` for a quick definition, `check my understanding` to switch to evaluator mode.

### 🔬 Feynman check — `/templates/feynman`

One-shot auditor. You paste a plain-language explanation (200–400 words, for a smart 14-year-old). The auditor classifies every sentence and returns a ranked gap report with diagnostic questions you probably can't yet answer. Reveals exactly where your model is broken.

## Architecture

Pragmatic, no backend database.

- **Astro 4 + React islands** for the UI shell, interactive components hydrate on demand.
- **Tailwind 3** with a hand-tuned `.prose` for typography (Charter serif for reading, mono for labels).
- **`marked`** for markdown rendering; two small parsers turn `roadmap.md` into week cards and `progress_log.md` into a timeline.
- **Local Claude Code CLI via a tiny Node bridge** (`ui/bridge/server.mjs`, port 4322) — spawns `claude -p --session-id <uuid> --output-format stream-json`, parses SSE, forwards text + usage back to the browser. No API key needed; uses your existing Claude subscription.
- **Anthropic API as fallback** via `@anthropic-ai/sdk` with `dangerouslyAllowBrowser: true`. Key lives in `localStorage`; XSS-warned.
- **Vite middleware dev fallback** — in dev mode, if no GitHub repo is configured, `/_local/*` reads and writes the parent filesystem, so the wizard's "Save to repo" writes markdown directly.
- **State in `localStorage`** — active sessions (refresh-safe, with pointer to the on-disk Claude Code session for `--resume`) + per-week study progress + cumulative token usage.

```
Browser (Astro + React)
    │
    ├─── /_local/* ──────────── Vite middleware (dev only) ─── parent filesystem
    ├─── api.github.com ─────── GitHub Contents API (prod)
    ├─── raw.githubusercontent ── markdown fetch (prod)
    │
    └─── POST /run ────── node bridge/server.mjs ──── claude -p ──── Claude session on disk
                      OR  fetch(api.anthropic.com)
```

## Configuration

### `ui/.env`

```env
PUBLIC_GITHUB_OWNER=your-username
PUBLIC_GITHUB_REPO=learnos
PUBLIC_GITHUB_BRANCH=main
```

Required for production builds. In dev, leave blank and the Vite middleware takes over.

### Model selector (top bar)

- **Claude Opus 4.7** (default) — best quality for Socratic and Feynman work.
- **Claude Sonnet 4.6** — faster / cheaper for roadmap drafts and casual chat.

### Backend toggle (top bar)

- **CLI** (default) — routes through the local `claude` CLI. Requires Claude Code installed and authenticated locally. No API key needed.
- **API** — direct browser → Anthropic. Requires a pasted key stored in `localStorage`.

### Token counter

Cumulative input / output / cache-read / cost across every run. Click for a breakdown; reset with the button.

## The manifesto

1. **Learning is shipped, not consumed.** A tutorial watched is not a skill owned. Close every module with a built artifact.
2. **Depth over surface area.** Four topics learned deeply in a year beat forty skimmed.
3. **First principles or bust.** If your explanation bottoms out in jargon, you don't understand it.
4. **The Feynman test is the only test.** You know a concept when you can teach it to a sharp 14-year-old with no jargon.
5. **Struggle is the signal.** The uncomfortable concept is the high-leverage one.
6. **Primary sources over aggregators.** Official docs, original papers, the one canonical book. **Claude is never a substitute for reading them.**
7. **Version everything.** Roadmaps, notes, dead ends, wrong models.
8. **Teach to close the loop.** A concept is not yours until you have explained it to another human.
9. **Respect the schedule, not the mood.** Motivation is weather; systems are climate.
10. **Retire ruthlessly.** Learning debt is real debt.

---

## Non-goals

- **No AI summarization of primary sources.** That violates the whole premise — we'd be replacing books with aggregators. Claude primes, audits, and drills. The reading stays with you.
- **No multi-user / no backend database.** This is a personal tool. State lives in your browser and your git repo.
- **No cloud deploy story baked in.** Build with `npm run build`, deploy the `dist/` to anywhere static. The CLI backend is local-only — production deployments use API mode.
